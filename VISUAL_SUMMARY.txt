┌────────────────────────────────────────────────────────────────────────────────┐
│  WHY THIS WORKS (ICA IS PERFECT FOR PARALLELIZATION)                          │
├────────────────────────────────────────────────────────────────────────────────┤
│                                                                                │
│  ✅ Independence                                                               │
│     Each subject's ICA is completely separate (no dependencies)              │
│                                                                                │
│  ✅ Computational Intensity                                                    │
│     - pop_runica: Eigenvalue decomposition (high CPU)                        │
│     - pop_iclabel: Deep learning inference (high CPU)                        │
│     - Results: Benefits from parallel execution                              │
│                                                                                │
│  ✅ Safe I/O                                                                   │
│     Each subject uses unique ID (Nc04, Nc05, etc.)                           │
│     Files named with subject: Nc04_ICA_pruned_June26.set                    │
│     → No file collisions between workers                                     │
│                                                                                │
│  ✅ No Shared State                                                            │
│     No variables modified across subjects                                    │
│     Each worker: load → process → save (independent cycle)                   │
│                                                                                │
